# KML Challenge 2026S 회고 — Public 1위 → Private 2위 (shake-up 패배)

> 고객 성별 이진분류(ROC-AUC). **Public 1위였으나 Private 2위로 shake-up 패배.** 다음 대회 반영용 회고.
> Public: 0.7354 → 0.7369(pseudo fold10) → **0.7385141(우리 public 최고)**.
> **Private: 우리 0.7324252(2위) vs 1등팀 0.7345426(1위) = −0.00212 패배.**
> ⚠️ **핵심 교훈은 섹션 17(shake-up 사후분석)에 있음 — 이 회고에서 가장 중요.** 우리의 fold-variance 돌파는 public 기준 화려했으나 private엔 거품이었고, 진짜 패인은 public-overfit이었다.

## 1. 협업 방식 (사용자 ↔ AI)
- **분업**: 사용자=방향·결정, AI=코드·검증·기록.
- **원칙(사용자가 못 박음)**: "절대 천장이라 하지 마", "다 시도해봐", "점수 나오면 즉시 기록(배치 금지)".
- **모든 실험 루프**: 가설 → 코드 → CV → corr → LB → 노션 즉시 기록. 감이 아니라 숫자로 결정.
- **외부 AI 교차검증**: GPT/Gemini 조언 + 우리 실측이 같은 걸 가리키면 채택, 한쪽 일반론은 거름.
- **팀 분담**: 사용자=pseudo 정제, 김민형=base/RunPod 재구축, 발표자료·코드 공유.

## 2. 봉착한 한계
- **0.7354 "데이터 천장" 오판** (며칠). ~25가지 시도 전부 흡수/미달:
  - pseudo 9변형 + 강화 7개(임계/topK/weight/co-training/deep iterative)
  - 임베딩(W2V/FastText/Doc2Vec), feature selection, SCARF, FT-Transformer
  - target encoding, adversarial validation, 125멤버 스택, transductive
  - 두 AI(GPT+Gemini) 제안 60피처(goodcd계층/transition/basket/drift/gender-interaction)
- **핵심 증상**: 새 피처가 단독은 직교(corr 0.3~0.7)인데 521+pseudo 블렌드에 넣으면 **corr 0.98~0.99 흡수**.
- **근본 원인 = "더하기 게임"**: 이미 포화된 base에 계속 쌓아서 노이즈만 추가.

## 3. 돌파 (어떻게 더 짜냈나)
- **결정적 한 방 = pseudo student fold 5→10 (+0.0013, 5위→1위).**
  - #92(5-fold pseudo, 0.7354) → #112(10-fold, 0.7367) → #120(K=50, 0.7369001 🥇)
  - 레시피: `rank(0.70·rank(#73) + 0.30·rank(pseudo_student_Kfold))`
- **왜 통했나**: pseudo는 노이즈 라벨(틀린 test행 hard label)을 학습 → fold 변동 큼. fold↑ = test 예측을 K개 모델 평균 → variance↓ + 각 모델 학습량↑(80%→90%). **일반 모델이면 안 나왔을 효과(pseudo 특수성).**
- **천장은 데이터가 아니라 fold-5 variance의 한계였음.**
- **2차 돌파 = base-fold (#73 자체를 10-fold로, +0.0015).** pseudo만 fold↑했던 걸, **base(#73, 블렌드 0.7비중)도 fold↑**:
  - #73(5f base, 0.73468) → #123(10f base, 0.7361546, **+0.0015**) → 0.7비중으로 전이 → **#126(0.7382929 🥇)**
  - 방법: #73 메타스택 ~110멤버 중 **재폴드 가능 27개**(GBM코어/optuna/mega/AG megaA621/interact/FS)만 10-fold판으로 교체(나머지 ~85개 5f import 유지) → 3-layer 스택.
  - **교훈**: variance 레버(fold↑)는 **블렌드 모든 축에 독립적으로 적용**됨. pseudo축만 짜고 base축은 5f로 방치했던 게 마지막 빈틈. "가장 큰 가중치(0.7) 컴포넌트부터 fold↑"가 EV 최대.

## 4. 검증으로 가짜 걸러냄 (발견만큼 중요)
- **leak 점검**: 신규 피처 lb_g6_skew가 전체 train으로 계산돼 +0.005를 가짜로 부풀린 것 적발 → KFold OOF로 수정하니 +0.00156로 축소.
- **OOF≠LB 구분**: K↑ 시 OOF +0.0014는 held-out 작아지는 아티팩트, 진짜 LB 이득은 +0.0001.
- **멀티시드 오염**: s42 재생성 CV 0.711(s7은 0.719 정상) = 버그. "멀티시드 무익"의 진짜 원인.
- **corr 기반 hedge 선정**: final 2슬롯을 #1위 대비 corr로 측정 → 73만 corr 0.997 독립.

| 10 | 0.7367072 | +0.0013 큰 점프 |

## 6. 다음 대회 — 효율 개선 (★핵심★)
| 이번 실수 | 다음엔 |

| 521피처 + 25가지 더하기 | **데이터 크기에 맞게** (5만 행 → 피처 100~150, 모델 3~4개) |

**가장 큰 비효율**: "더 정교한 모델"이 아니라 **"이미 만든 것의 variance를 줄이는 단순 레버(fold↑)"가 답**이었는데 며칠 늦게 발견. 복잡도 추가 전에 분산 감소부터.

## 7. 핵심 교훈 (7가지)
1. **"더하기"보다 "안정화"**. 포화 상태에선 피처/모델 추가 ❌ → variance 감소(fold/seed) ✅. 상위팀도 "치환·선별"로 이김.
2. **데이터 크기가 모든 걸 정한다**. 5만 행엔 521피처 과포화. 처음부터 크기에 맞춰 설계.
3. **"천장"은 거의 항상 가설**. 우리 천장은 fold-5 한계. 단정 전 안 건드린 축 의심.
4. **CV≠LB 절대 잊지 마**. OOF +0.0014 = LB +0.0001(아티팩트) 빈번. LB로만 확정.
5. **검증이 발견만큼 중요**. leak 점검(가짜 +0.005 적발), corr hedge — 가짜에 안 속게.
6. **외부 AI는 교차검증용**. GPT+Gemini+실측이 일치할 때만 신뢰.
7. **즉시 기록 = source of truth**. 수십 실험 재검증 불가 → 기록이 유일한 원본.

## 8. 딥리서치 · K-fold 운영 전략

### 8-1. 대회·데이터 맞춤 딥리서치를 계속 돌려라
- 막힐 때마다 **"이 대회·이 데이터 형식" 특화 딥리서치** 반복이 효과적.
- 일반론 ❌ → **우리 스키마 + 이미 한 것을 다 명시**하고 "안 한 것 중 EV 높은 것"만 받기.
- 유사 공개대회(L.POINT/Tianchi/KDD repeat-buyer) + 작년 1등 노트북 **대조**로 빠뜨린 것 검증.
- 딥리서치 결과 = 외부 AI 조언과 또 교차검증. 셋 다 일치하면 신뢰.

### 8-2. 데이터 수에 따라 K를 적절히
- **K는 데이터 크기 종속.** train 3만 기준 fold당 학습량: K5=80%/K10=90%/K15=93%/K20=95%.
- 5→10 +10%p로 점프(+0.0013), 10→20 +5%p라 미미. 분산 `Var≈σ²(ρ+(1-ρ)/K)` — ρ 높으면 K 키워도 포화.
- **실전(3만~5만 행)**: 빠른실험 K5 / 최종 K10 / 추가 K15 / 여유 K20. K30+ 비추.
- 대형(수십만+)이면 K↑ OK, 소형이면 K10 sweet spot. **데이터 크기 먼저 보고 K.**

### 8-3. ★K를 막판 천장 돌파용으로★ (핵심)
- 초반 K=5로 빠르게 탐색 → **천장 느낄 때 K↑ 테스트** = 1위 결정타.
- K↑ = 새 신호 아니라 variance 감소 → "더 짤 게 없을 때" 남은 깨끗한 레버.
- pseudo처럼 fold 변동 큰 모델에서 효과 큼.
- ⚠️ K↑는 OOF 아티팩트로 부풀려짐 → LB로만 확정.

## 9. ⚠️ AI 협업 주의점 (실제 겪은 약점 + 해결책)

### 9-1. AI가 Phase 1/2/3 계획해놓고 1만 하고 넘어감
- 증상: "①cat10f ②멀티시드 ③15f" 해놓고 ①만, ②③ 흐지부지. "2번도 했어?" 물어야 함.
- 해결: ①체크리스트는 **사용자가 직접** ②"전부 했어?" 주기적 점검 ③AI에게 "①✅②⬜③⬜" 상태표 매 턴 출력 ④**한 번에 한 단계씩** 명령 ⑤세션 끝 "한 것 vs 계획" 회수.

### 9-2. AI 다른 약점
- OOF 흥분 → "LB 확정했어?" / 직교성만 보고 좋다 함 → "CV 강도는?" / 위로성 포장 → 팩트체크 압박 / 검증없이 추정 → "넣기 전 확인했어?".

### 9-3. ⚠️ 긴 세션 할루시네이션
- 증상1: 이미 한 실패 실험을 "안 해봤죠? EV 높아요"로 재제안. 증상2: 지적하면 근거없이 말 바꿈(영합).
- 해결: ①**기록을 진실의 원천으로**("했는지 노션/파일 먼저 확인") ②"코드/파일로 증명해"(`ls artifacts/oof`) ③말 바꾸면 "근거 대" 압박 ④**핸드오프 문서로 컨텍스트 압축 후 재시작** ⑤중요 결정은 "숫자로 다시 보여줘"(기억 의존 X).

### 9-4. ★타 AI 교차검증 적극 활용★
- 단일 AI는 자기 컨텍스트에 갇혀 같은 실수 반복 → 다른 AI는 그 편향 없음.
- 세 정보원 일치 = 신뢰도 급상승(GPT+Gemini+실측이 "fold 진짜, K 포화" 일치).
- 방법: 상황 프롬프트 여러 AI 동시 + "우리가 한 것" 명시(중복방지) + 표로 합의/불일치 + 메인 AI에 "외부답+실측 교차검증해". 최종 판정은 **항상 LB**.

### 9-5. ★"거짓 천장 단정" 끓어내기★ (최대 교훈)
- 실제 0.7354를 며칠 "데이터 천장" 단정 → fold-5 한계였음. 가장 비싼 실수.
- 왜: "다 실패"="더 할 거 없다" 비약 + 안 건드린 축(학습 절차) 놓침 + 확증편향.
- 끓어내기: ①**"천장" 금지 → "건드린 축의 한계"로만** ②"안 건드린 축 나열"(피처/모델만 아니라 **fold/seed/bagging/early-stop/CV방식/앙상블구조**) ③"실패 vs 잘못 적용" 구분("틀린 방식 아니야?") ④variance 레버는 **마지막에 반드시** ⑤외부 AI에 "진짜 천장? 안 본 축?" 명시 질문 ⑥**사용자 "정말 없어?"를 "미탐색 축 재탐색" 신호로**.

## 12. ⚠️ EDA 보완점 (이번에 아쉬웠던 부분)
**EDA 부실이 "거짓 천장"의 근본 원인 중 하나.** 데이터를 충분히 안 보고 바로 521피처+모델링으로 들어가서, "무엇이 진짜 신호인지" 모른 채 더하기만 함.

### 12-1. 이번 EDA의 구멍
- **타겟 vs 각 변수 관계를 체계적으로 안 봄** → 피처를 "감"으로 만들고 흡수 여부를 사후 확인.
- **데이터 크기 대비 피처 수 감각 없음** → 3만 행에 521피처 과한 걸 EDA에서 알았어야.
- **train/test 분포 비교(adversarial)를 늦게** → 처음에 했으면 CV 신뢰도·pseudo 작동 이유를 일찍 앎.
- **goodcd 13자리 계층 구조 늦게 파악.**
- **★fold별 분포·안정성 안 봄★** — 결정적. fold마다 AUC 편차 큰 걸 EDA에서 봤으면 "fold=variance 레버"를 일찍 알아 천장 며칠 단축.

### 12-2. 다음 대회 EDA 체크리스트 (모델링 전 필수)
1. **타겟 분포** — 클래스 비율/불균형.
2. **타겟 vs 각 변수** — 범주형=카테고리별 타겟비율(+표본수), 수치형=분위수별 타겟률. **각 변수 단독 판별력(AUC/상관) 표로** → 진짜 신호 식별.
3. **데이터 크기 vs 피처 수 비율** — 작으면 처음부터 피처 적게.
4. **결측·이상치·인코딩** — cp949/한글, 음수금액(환불), junk 코드.
5. **고카디널리티 구조** — goodcd 계층(prefix)/카디널리티/빈도 → TE·임베딩 전략 결정.
6. **시간 구조** — 시간/기간/시즌성, train-test 시간분리(시간누수).
7. **train vs test 분포(adversarial)** — **초반에.** AUC≈0.5면 CV 신뢰, 높으면 공변량이동→CV 수정.
8. **세그먼트별** — 활동량 적은/많은 고객, cold-start 비율 → CV 설계 반영.
9. **★fold별 안정성★** — 같은 모델 여러 fold/seed로 **AUC 편차(std) 확인.** 편차 크면 **variance 레버 살아있는 신호**(fold↑) → 이번에 봤으면 fold-10 일찍 발견.
10. **상관·중복 피처 지도** — corr 히트맵, 중복 많으면 처음부터 정제.

### 12-3. EDA → 설계 연결 (핵심)
- EDA는 "예쁜 그래프"가 아니라 **"피처 설계·모델 선택·CV 전략·K 설정의 근거".**
- 이번에 빠진 **(2)단독 판별력 + (3)크기/피처 비율 + (9)fold 안정성** 3개가 며칠 헤맨 원인:
  - (2) 봤으면 → 흡수될 피처 덜 만들었을 것.
  - (3) 봤으면 → 521 과포화 처음부터 피했을 것.
  - (9) 봤으면 → fold-variance 레버 일찍 발견(천장 단축).

> **EDA 부실이 거짓 천장을 만들었다. 모델링 전에 ①타겟vs변수 단독판별력 ②데이터/피처 비율 ③fold별 AUC 편차(variance레버 신호) ④train/test 분포 를 반드시 정량화. EDA는 그래프가 아니라 설계 근거다.**

## 16. ★앙상블 아키텍처 (정확한 레시피 — 1위의 실제 구조)★
> **이 대회의 승부처는 FE/단일모델이 아니라 "앙상블을 얼마나 깊게 쌓았나"였음.** 개별 모델은 다 0.71대(도토리 키재기)인데, 4단계로 쌓아서 0.71→0.738로 +0.027 만듦. 정확한 구조 기록(다음 대회 그대로 재사용).

### 단계별 (개별 0.71 → 최종 0.738)
| 단계 | 구조 | OOF/LB | 게인 |
|---|---|---|---|
| **0. 개별 base 멤버 (~120개)** | 521FE first_xgb/lgbm/cat, mega(572FE) GBDT, AutoGluon(megaA/megamax), interact, et, optuna 등 | 단일 0.713~0.719 | — |
| **1. L1 메타스택 (7메타+hillclimb)** | base 풀(~120)을 LR(C=0.05/0.3/1)·Ridge·ExtraTrees(400)·HGB(300)·KNN(64)로 5-fold 스택 + hillclimb → L1 출력 8개 | — | base→메타 |
| **2. L2 hillclimb** | L1 출력 8개를 다시 hillclimb(greedy, 60iter) 결합 = **kim73/#73** | OOF 0.728 / LB 0.7347 | +0.01 |
| **3. fold10 base (#123)** | #73 풀의 재폴드 가능 27멤버를 10-fold판으로 교체 후 동일 스택 | LB 0.7362 | **+0.0015** |
| **4. pseudo 블렌드 (#126)** | `rank(0.70·rank(#123) + 0.30·rank(pseudo_50f))` | **LB 0.7383** 🥇 | +0.0014 |

### 핵심 메커니즘 (왜 이 구조가 통했나)
- **개별 모델은 corr 0.97~1.0으로 수렴** → 단일 모델 추가는 무의미. **메타스택(L1 7종 메타러너)이 서로 다른 결합방식으로 다양성 추출**이 핵심. (선형 LR/Ridge + 트리 ET/HGB + 거리 KNN + greedy hillclimb = 결합방식 다양화)
- **hillclimb(Caruana greedy)이 핵심 메타러너**: 약하거나 해로운 멤버를 자동 weight≈0 처리 → mega 단독 0.70인데도 스택이 알아서 적게 씀(과대평가 멤버 자동 정리). **그래서 mega 빼도 base corr 0.9996 안 변함(레버 아님).**
- **rank 정규화로 모든 결합**(rankdata/N): ROC-AUC=순위문제라 확률 캘리브레이션 불필요, rank 블렌딩이 자연스러움.
- **pseudo는 base와 독립 파이프라인**: teacher(student계열 격리) → test 고신뢰(≥0.9/≤0.1) 라벨 → student 재학습(10/50 fold). base 메타스택과 다른 축이라 0.3 비중으로 +0.0014 전이.
- **fold10은 base/pseudo 양쪽에 독립 적용**: variance 감소 레버라 메타스택 깊이와 무관하게 추가 게인.

### 다음 대회 재사용 체크리스트
1. 개별 모델 ~10~20개(GBDT 3종 + AG + 다양한 FE) → corr 수렴 확인.
2. **L1 = 결합방식 다른 메타러너 5~7종**(선형·트리·거리·greedy)으로 5-fold 스택.
3. **L2 = hillclimb**(과대평가 멤버 자동 정리, robust).
4. pseudo(test-side) 독립 생성 → rank 0.7:0.3 블렌드.
5. **천장 느끼면 fold↑**(base·pseudo 양쪽, 가중치 큰 것부터).
6. 전부 **rank 정규화 + 고정 folds + custid 정렬 통일**(스택 누수 방지).

## ★17. 왜 Public 1위인데 Private 2위로 졌나 — shake-up 사후분석 (이 회고에서 가장 중요)★
> **결론: final 선택 실수가 아니라, 우리 모델 자체가 1등팀보다 public에 과적합한 구조였다. final을 완벽히 골랐어도(private 최고 0.7326) 1등팀(0.7345)에 졌다.**
>
> 📊 **이 섹션의 그래프 버전 → [`shakeup-analysis.md`](shakeup-analysis.md)** (제출별 궤적 · K-fold 전이 실패 · 팀별 낙폭 비교)

### 최종 성적 (public → private)
| 팀 | public | private | 갭 | 갭-public 상관 r | 순위변동 |
|---|---|---|---|---|---|
| 1등팀(우승) | 0.7378695 | **0.7345426** | −0.0033 | **−0.19** (robust) | 2위→**1위** |
| **우리** | **0.7385141** | 0.7324252 | **−0.0061** | **+0.63** (과적합) | **1위→2위** |
| 3등팀(47제출) | 0.7319068 | 0.7323242 | **+0.0004** | −0.36 | 7위→**3위** |
| 4등팀 | 0.7369901 | 0.7296843 | −0.0073 | +0.44 | 3위→4위 |

### 진짜 원인 (교수님 LB분석 + 우리 데이터 일치)
1. **갭-public 상관 r = +0.63 (우리) vs −0.19 (1등팀)** — 우리는 **public이 높을수록 갭이 커지는** 전형적 LB 과적합. 1등팀은 public 올라도 갭 안 늘었다. **같은 public 0.737~0.738 구간의 갭 차이 ~0.003이 그대로 역전.**
2. **초기 적합이 출발점을 결정** — 베이스 모델이 처음부터 public/private 어디에 fit하느냐. 우리는 **처음부터 public-fit(+0.0043)**이었고 한 번도 못 뒤집음. 이후 모든 블렌드·pseudo·fold튜닝이 그 public-fit 베이스를 상속.
3. **운영이 갭을 증폭** — 우리는 110+제출로 public 피드백 좇으며 갭 2배(+0.0043→+0.0071 수준). 3등팀(47제출)는 갭 +0.0004로 거의 0 = private 3위. **제출 수 = 과적합 지표.**
4. **pseudo + fold↑가 public 거품의 주범** — 우리 돌파(0.7367→0.7385)는 public 기준 화려했으나 **private엔 −0.006 거품**. fold-variance 감소는 "public variance"를 줄인 거라 private 전이가 작았다(GPT 예측 +0.0010~0.0018보다도 적게 전이).
5. **final 선택은 부차적** — 우리 Slot1(#126)=private 14위, Slot2(#123)=private 19위(헤지가 본진보다 나쁨). 하지만 **private 최고(151=0.7326) 골랐어도 1등팀(0.7345)에 −0.0019로 졌다.** 풀 자체가 문제.

### ★다음 대회 액션 (교수님 3교훈 + 우리)★
1. **중반부터 "CV − Public 갭 추이"를 그려라.** 후반에 갭 띠가 두꺼워지면 **public 튜닝 즉시 중단** 신호. (우리는 이 신호를 무시하고 끝까지 긁었다)
2. **제출 수를 통제하라.** public LB를 validation처럼 쓰면 과적합. 3등팀 47제출 private 3위 vs 우리 140제출 private 2위.
3. **저상관 독립 계열을 미리 확보하라.** 점수 비슷하면 corr 낮은 걸 final에 넣어 헤지. 우리는 73기반 단일 계열로 수렴해 진짜 헤지가 없었다(#123도 같은 base-fold 계열).
4. **새 신호(FE) > 가중치 튜닝.** 단순 FE(AutoGluon→W2V→FastText) 4단계만으로 private 4등 가능했고 **private이 public을 줄곧 앞섰다(=진짜 신호)**. "좋은 피처 하나가 가중치 튜닝 수십 번보다 컸다." → 우리 섹션 11(EDA부실) 교훈과 직결.
5. **팀 간 다양성 = 마지막 무료 점심.** 팀 내 corr 0.99+ 고갈돼도 팀 간 corr 0.92~0.97. (대회 내 활용 불가지만, "다양성 고갈 = 천장"의 진짜 의미)
6. **final은 절대 public-max 금지** — 우리가 며칠 외쳤으나 결국 #126(public 1위)을 골랐다. CV robust + 저상관 우선.

### 한 줄
> **public-overfit은 마지막 날 선택이 아니라 "베이스 모델을 만드는 과정 전체"에서 결정된다. 운으로 받은 초기 적합(출발점)은 못 바꿔도, 거기에 거품을 쌓을지(public 튜닝)는 선택이다. 우리는 fold-variance 돌파에 취해 public 거품을 110번 쌓았고, 1등팀은 robust 베이스를 지켰다. 다음엔 갭 추이를 그리고, FE로 진짜 신호를 더하고, 제출을 아껴라.**

## ★18. 1·3등은 실제로 무엇을 했나 (우리와 정반대 = 다음 대회 청사진)★
> **1·3등 발표자료 분석. 우리 public 절대점수(0.7385)는 둘보다 높았으나(1등팀 0.7379, 3등팀 0.7319), robust 장치가 없어 private에서 거품이 빠졌다. 핵심: 둘 다 pseudo 안 쓰고, CV/drift로 public 과적합을 능동 억제했다.**

### 1등 1등팀 — drift 제한 + public 과적합 능동 억제
- **EDA로 feature 방향 결정** → feature selection(top200/300/800, stable importance, group quota)
- **drift-aware selection (핵심 private 방어)**: feature 선정 시 성별 신호(strength)뿐 아니라 **train/test 분포차(drift)도 제한**(DRIFT_MAX=0.35). drift 크면 제외.
- **균등 rank로 public 과적합 능동 억제**: `private_safe = 0.75·opt_rank + 0.25·equal_rank`
- **public 최고(v12 32.5%)보다 보수적(30%) 선택** → private 보상
- **pseudo-labeling 안 씀.** W2V/FastText는 보완축, rank01 blend, 4명 팀 다양성(corr 0.92~0.97)

### 3등 3등팀 — CV-LB Gap을 신앙처럼 (47제출)
- **모든 실험에 CV·LB·Gap 기록.** Gap −0.011~−0.014로 **일정한지**가 robust 신호
- **stacking_4는 CV 0.756(자기 최고)였으나 Gap +0.041로 뒤집혀 제외** — CV 최고를 과적합으로 판단해 **버렸다**
- **"LB 최적화보다 CV 기반 일반화 우선"** 명시. 4관점 분리 FE(OOF 상관 0.577) → AutoGluon 2단계 stacking. pseudo 안 씀.

### 우리(2등) vs 1·3등 대조표
| 항목 | 우리 | 1등팀(1등) | 3등팀(3등) |
|---|---|---|---|
| CV-LB Gap 관리 | ❌ public만 | drift 제한 | ✅ 전실험 Gap 기록 |
| public 과적합 억제 | ❌ public 좇음 | ✅ 균등rank 25% | ✅ CV우선 |
| CV 최고 처리 | public 최고 채택 | — | ✅ Gap 크면 버림 |
| feature | 521+pseudo 누적 | ✅ top-N pruning | ✅ CV개선시만 제거 |
| pseudo-labeling | ⚠️ 핵심 레버 | 안 씀 | 안 씀 |
| 제출 수 | 140 | 131 | 47 |

### ★다음 대회 구체적 청사진 (1·3등에서 직접 베낄 것)★
1. **CV·LB·Gap을 모든 제출에 표로 기록.** Gap이 일정하면 robust, +로 뒤집히면 과적합 → 그 제출 버려라(3등팀 stacking_4). **CV 최고 ≠ 채택.**
2. **drift-aware feature selection**: feature 고를 때 strength(성별신호) + train/test drift 동시 제한. drift 큰 feature는 성능 좋아 보여도 제외(1등팀).
3. **균등 rank를 섞어 public 과적합 능동 억제**: `0.75·opt_rank + 0.25·equal_rank`. final은 public 최고보다 1~2단계 보수적으로.
4. **feature는 쌓지 말고 pruning**: top-N selection, 제거 후 CV 개선될 때만 반영. 521개 누적 = 노이즈.
5. **pseudo-labeling 신중/회피**: 우리 public-overfit의 주범. test-side 신호라 public에 fit. 1·3등 둘 다 안 썼다. 쓸 거면 비중 낮추고 drift 검증.
6. **제출 수 통제**: 47제출(3등팀 3위) < 140제출(우리 2위). 적게 내고 CV 신뢰.

### 한 줄
> **우리는 "점수 최대화"(pseudo+fold+110제출)에 올인했고, 1·3등은 "robust 최대화"(CV-LB gap 관리·drift 제한·feature pruning·pseudo 회피)에 집중했다. 화력은 우리가 셌지만 private는 robust가 이겼다. 다음엔 점수가 아니라 Gap을 관리하라.**

## ★19. 교훈의 적용 범위 — 이 대회 특수조건 vs 범용 원칙 (오적용 방지)★
> **우리가 얻은 교훈이 모든 대회에서 통하는 게 아니다. "이 데이터였기 때문에 맞았던 것"과 "어느 대회나 맞는 것"을 나눈다. 다음 대회 시작 시 섹션 0의 프로파일과 대조해서 A는 조건부로만, B는 항상 적용할 것.**

### A. 이 대회 특수조건에서만 맞았던 교훈 (조건 다르면 뒤집힘)
| 우리 교훈 | 맞았던 이유(이 대회 조건) | 조건이 바뀌면 |
|---|---|---|
| **pseudo-labeling은 독약** | test 작고 **public-private 분포차 큼** → pseudo가 public-fit | **분포 잘 맞는 대회(corr 0.97+)면 pseudo가 진짜 이득.** adversarial AUC 낮으면 적극 써도 됨 |
| **feature는 쌓지 말고 pruning** | **30k행에 521피처 = 과포화** | 데이터 크면(수십만+) 피처 더 쌓아도 됨. pruning은 작은 데이터 처방 |
| **fold↑ variance가 핵심 레버** | 작은 데이터 + 모델 corr 수렴 | 큰 데이터·미수렴이면 fold↑ 효과 작음. variance보다 bias가 문제일 수 있음 |
| **앙상블/스택이 승부처** | 개별 모델 천장 0.71(신호 약함) | 신호 강한 대회(단일 0.9+)면 단일 모델·FE가 더 중요, 스택은 소폭 |
| **균등rank 25% 섞기(1등팀)** | **public-private shift 큼** | shift 없으면(corr 0.97+) public-opt rank 그대로가 더 나음 |
| **drift-aware selection(1등팀)** | train/test 분포차 존재 | 분포 동일하면 drift 제한이 오히려 좋은 feature 버림 |
| **rank 블렌딩** | **AUC(순위 지표)** | LogLoss/Brier면 확률 캘리브레이션이 중요, rank만으론 부족 |

### B. 범용 원칙 (어느 대회나 항상 — 데이터 무관)
1. **CV≠LB.** OOF 부풀림(held-out 작아짐·stacking amplification) 항상 의심, 최종 판정은 LB/private.
2. **CV−LB Gap을 모니터링하라.** Gap이 일정=robust, 벌어지면 과적합. (이 대회 3등팀 핵심, 모든 대회 유효)
3. **final은 public-max 금지.** public 1위 함정. CV robust + 저상관 헤지 우선.
4. **저상관 독립 계열을 헤지로 확보.** 단, 헤지도 "독립 + 약하지 않은 것"이어야(우리 #123 실패).
5. **"왜 오르는지" 메커니즘을 설명할 수 있어야.** 설명 안 되면 mirage 위험(OOF 순환 과적합).
6. **"천장" 단정 말고 안 건드린 축(학습 절차 포함) 끝까지.** 단 천장 돌파도 public 거품일 수 있으니 LB로 검증.
7. **실패·결과 즉시 기록(왜 실패했나 메커니즘까지).** 수십 실험 재검증 불가, 기록이 유일 원본.
8. **누수 검증**(leak/adversarial/fold 정렬)은 항상.
9. **제출권은 점수 갱신이 아니라 "진단/정보 획득"에 써라.** corr 0.999 복사본 제출 = 낭비.

### 다음 대회 시작 시 체크 (섹션 0 프로파일 채우고 → 분기)
1. **adversarial validation 먼저** → train/test AUC. **0.5 근처(분포 같음)면**: pseudo OK, public 더 신뢰 가능, drift 제한 불필요. **높으면(분포 다름)**: 이 대회와 같은 처방(robust·drift·균등rank·pseudo 회피).
2. **데이터 크기** → 작으면(수만) pruning·fold↑, 크면 피처 더·단순 CV.
3. **개별 모델 천장** → 낮으면 앙상블 승부, 높으면 FE·단일 승부.
4. **지표** → AUC면 rank, LogLoss면 calibration.

### 한 줄
> **우리 교훈의 절반(pseudo독약·pruning·fold↑·균등rank)은 "작고 과포화에 public-private 분포차 큰 이 대회" 때문이었다. 범용은 B의 9개뿐. 다음 대회는 adversarial validation으로 분포차부터 재고, 같으면 A를 뒤집어 적용하라.**
