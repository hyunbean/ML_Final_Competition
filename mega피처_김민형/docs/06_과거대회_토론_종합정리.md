## 2. ⚠️ 0.81의 정체 (kml2020/2020f) — "다른 게임"
kml2020f 토론(작성=교수 본인) 원문 요약:
- **"How to achieve the baseline"**: Baseline = **Power Mean( Group Project #2의 1·2·3등 + DNN-Only + Conv1D + DAE-DNN )** with p=3.5. New Baseline = PM( Old Baseline, PM(Group Project #1 우승작들), PM(W2V Individual Project 우승작들) ).
- **"Ensemble performance" 표**: (((4+5)+3)+2)+1 조합 → AUC **0.808584**, p-value 11.5 → **0.808744**.
- → **0.81은 단일/한 팀 성과가 아니라, 한 학기 전체 그룹프로젝트·개인프로젝트 우승작 수십 개를 Power Mean으로 누적 앙상블한 "수업 통합 베이스라인"**. 게다가 데이터가 raw 거래가 아니라 W2V/사전집계본 = **태스크·정보량 자체가 다름.**
- **우리(4일·한 팀·raw 거래)가 복제 불가능한 구조.** 0.81을 우리 천장의 반례로 볼 수 없음.

## 3. 🎯 교수가 직접 가르친 "우승 기술" (토론에서 추출 — 우리가 바로 쓸 것)
kml2019·kml2020f·mba 토론에 교수가 반복해서 올린 핵심 전략:
### (A) PL-score "산 능선(mountain ridge)" 앙상블 선택 — `yhcho1918/ensemble-strategy-using-public-leaderboard`
- Santander 2등 솔루션 차용. **x축=평균 상관도, y축=Public LB 점수** 산점도에서 **능선(고LB·저상관) 위의 제출들만** 최종 앙상블에 선택.
- "It is desirable to pick submissions from the mountain ridge." → **다양성+고성능 동시 만족하는 제출만 고르라**.
- ✅ 우리는 **40개 LB 실측 제출** 보유 → 이 전략 그대로 적용 가능.
### (B) Power Mean 앙상블 (p 튜닝)
- "ensemble low-score predictions first, then re-ensemble with high-score ones."
- "When PM-ensembling high-score submissions, use higher p-value" (p=3.5→11.5일수록 고득점에 유리).
- 산술평균이 아니라 **Power Mean(거듭제곱평균)**, p를 키우면 고신뢰 예측 강조.
### (C) 최종 제출 2개 선택 (mba 토론)
- "복수(2개)로 선택하는게 유리" — Public 최고 1개 + 직교적 1개.

## 4. 과거 우승 솔루션 접근법 (mba 커널·토론, 우리와 동일 확인)
- **caffein123 (1130-final, 최다추천)**: 평균할인율·남성파트비율·화장품비율·쇼핑체류시간·브랜드별구매 → **GBC+XGB voting + NN, 다시 앙상블**.
- **Marvel**: 2버전 FE × 다른 모델 → **geometric ensembling**. 피처에 "여성브랜드구매점수/고성별연관제품/여성제품점수" 등 성별-특정 점수.
- **채범주희**: 요일 월화수/목금/토/일 분리, 계절 1개월 당겨 분기, GBC+XGB voting.
- 교수 "Features you can consider": **RF baseline AUC 0.6835/0.6789** (출발점).
- 앙상블 참조: Otto-group challenge discussion/14335.
- → **전부 우리가 한 것(집계FE+TE+voting+NN+앙상블)과 동일. 천장 0.73 재확인.**
