## 해석 — 3개 숫자의 충돌
| 항목 | 값 |
|---|--:|
| 우리→1등 갭 (0.73203 − 0.72964) | **+0.00239** |
| public 낙관편향 (public이 부풀려보임) | **+0.00256** |
| private 변동성 (셰이크 std) | **±0.00189** |

- **갭 < 낙관편향**: public에서 1등을 넘겨도, 그 점수가 +0.0026 부풀려진 거라 **private에선 사라질 수 있음.**
- **갭 ≈ 노이즈**: 1등 추월이 노이즈 1.3배 안 = **사실상 동전던지기.** private 1등은 주로 운.
- → **public LB를 더 짜내 1등 넘기는 통계적으로 신뢰 불가.** 공격적 public 튜닝은 private서 **2등→3등 추락(셰이크) 위험.**

## 좋은 소식 — 선택 과적합은 작다
- **public-선택 후회 +0.0002뿐** → 능선/Caruana로 골라도 honest와 거의 동급. **세션1 능선 작업 자체는 안전.**
- 위험은 "선택 방법"이 아니라 **"public 절대점수를 믿고 공격적으로 거는 것"**.

## 최종 2픽 권고 (시뮬 근거)
private가 노이즈 지배 → **공격보다 robust 헤지:**
1. **① 메인 = honest-OOF 최강 (42_nestedCaruana)** — full-OOF 선택이 oracle과 거의 동급(0.72109 vs 0.72112), private 기대 최고·안정.
2. **② 헤지 = public 최고 (24_hillclimb 0.72964 또는 세션1 능선 산출물)** — 공격 1장.
- ⚠️ **public에서 1등 넘는 제출이 나와도 2장을 거기 몰지 말 것.** +0.0026 낙관편향 안이면 private 보장 X.
