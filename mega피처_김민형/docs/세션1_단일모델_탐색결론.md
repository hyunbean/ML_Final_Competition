## 🔮 합성데이터(LLM) — ⚠️ GBT/AG 말고 **딥러닝 모델에서만** 시도할 것 (명시)
> **결정사항: LLM 합성데이터·합성정보는 tabular GBT/AutoGluon에 쓰지 말 것. 딥러닝(시퀀스·임베딩·SSL) 모델 쪽에서만 실험.**
- **tabular 단일(GBT/AG)엔 무익 — 검증완료**: 선물고객 40명서 LLM추론 0.56 << 모델 0.80. GBT는 572피처+학습된 선물패턴으로 표면추론을 압도 → 합성 라벨/정보가 노이즈. **GBT/AG엔 투입 금지.**
- **그러나 딥러닝 모델 쪽은 다를 수 있음** (사용자 제안, 타당):
  - 딥(시퀀스/임베딩/탭ular-NN)은 데이터가 적어 **데이터증강에 민감** → LLM 생성 구매시퀀스/페르소나가 **사전학습/증강 신호**로 쓰일 여지.
  - Simula식 reasoning-trace를 **딥 모델의 보조 입력/distillation 타깃**으로 (GBT처럼 즉시 피처가 아니라).
  - **권장 실험**: 딥 모델(seq/SCARF/FT-Transformer) 학습 시 LLM 합성 시퀀스로 pretrain → real fine-tune. 단독 단일은 약해도 **앙상블 직교멤버**로서 가치 가능.
