# 🧠 KML 2026S 성별예측 — FE 블록별 "출발 가설" 전체 정리

> 모든 블록의 **"어떤 생각에서 이 피처를 만들었나"** 를 사람 언어로. (s2_공유/blocks + session1_FE/blocks)
> 대전제: **라벨 = 결제한 카드 주인의 성별** (상품이 향하는 성별이 아님). → 취향은 뒤집힐 수 있음(선물).
> 표기: ⭐=유효(살아남은 신호), ≈0=포화/평탄. Δ는 mega/megaA 위 단일증분(있는 경우).

---

## A. 취향 = 성별 (무엇을 사는가) — 가장 기본 신호(corner 단독 0.66)
- **taxonomy** ⭐(+0.0008): "여성은 다양한 코너를 탐색, 남성은 소수에 집중" → 택소노미 7레벨 종류수·엔트로피.
- **domcat**: "각 고객의 대표(주력) 카테고리가 성별을 말한다" → corner/pc/brd/part 주력의 성별 log-odds·점유율.
- **catrecency**: "성별 상징 카테고리(화장품·란제리·남성복·골프·여성복)를 최근·자주 사는가" → 카테고리별 recency·count.
- **lifestage**: "구매 조합이 생애단계를 드러낸다(아동복∧완구=부모, 란제리=여성)" → 상징 카테고리 비율 + 부모 조합.
- **decisive**: "한 방에 성별 가르는 상품이 있다(브라=여성)" → 결정적 성별상품 최대 log-odds·부호.
- **wallet**: "여성은 지갑을 여러 카테고리에 펼치고 남성은 집중한다" → 카테고리 침투율(pen)·집중도(HHI).
- **brandport**: "브랜드 충성/다양성이 성별 다르다" → 브랜드 수·집중(HHI)·반복.
- **niche**: "대중적 취향 vs 희소·마니아 취향" → 브랜드/상품 대중성(인기도)·희귀상품 비율.

## B. 선물·가족카드 '역전/충돌' (라벨=카드주인이라 생기는 부작용) — 핵심 난제
- **conflict**: "남편카드면 레벨마다 성별이 엇갈린다" → corner/pc/brd 성별신호 불일치도.
- **dispersion**: "가족카드는 거래별 성별점수가 양극(이봉)이다" → 성별점수 분포 std/skew/이봉성.
- **goodcddisp**: 위와 동일하되 상품(goodcd) 레벨 분포 모양.
- **det**: "본인 취향은 반복구매, 선물은 일회성" → 반복(rep) vs 일회성(once) 성별 log-odds 분리.
- **gexcl**: "확실히 한쪽 성별만 쓰는 배타적 상품 소비량" → 성별 배타상품 카운트·방향·반복.
- **crossmulti** ⭐: "습관과 **반대방향** 구매(=선물)는 취향을 거꾸로 읽어야" → corner/pc/brd 다레벨 교차·sharp(고가·일회성·시즌). *(세션1 cross의 다레벨판)*
- **layered**: "선물 비중이 높으면 취향을 보정하라" → 습관취향 vs 선물취향 분리 + gate 보정값.
- **dualaxis**: "취향예측과 행동예측이 엇갈리면 그 사람이 선물고객" → 취향예측 vs 행동예측 차이 + 선물확신도.
- **returns**: "환불 행동(반품률·반품액)이 성별 다르다" → 환불 횟수·비율·대형환불.

## C. 가격 · 할인 · 할부 (어떻게 쓰나)
- **pricecat** ⭐(+0.0007): "카테고리별 가격민감도가 성별 다르다(의류는 정가, 식품은 알뜰 등)" → 팀(의류/잡화/식품)별 정가율·객단가·할인율.
- **pricepos**: "같은 코너 안에서 프리미엄 vs 알뜰 위치" → 코너내 가격 백분위·프리미엄/저가 비율.
- **spendshape**: "지출 분포의 모양(편중·불균형)이 성별 신호" → 지출 gini·cv·skew·분위·집중.
- **grain**: "상품 단가 입자(저가~고가 구간)별 성별성" → 단가 세분 성별 점수(금액가중).

## D. 시간 · 계절 · 주기 (언제 쓰나)
- **season** ⭐: "선물 타이밍(발렌타인·가정의달·연말)이 성별 신호" → 월별 지출 + 시즌 윈도우.
- **monthprof**: 월별 지출 프로파일(계절 소비 패턴).
- **periodicity**: "정기적(구독형) vs 충동적 소비" → 방문 주기·요일 엔트로피·규칙성.
- **timewin**: "라이프스테이지가 시간에 따라 변한다" → 분기별 성별점수·지출비중·추세.
- **trend / velocity**: "구매가 늘고 있나 줄고 있나" → 지출 기울기·활동개월·전후반 비율.
- **tastetime**: "시간대마다 사는 취향이 다르다" → 취향×시간 결합 임베딩.

## E. 점포 · 동선 · 바스켓 (어디서·어떻게 다니나) — 행동 지문
- **store**: "어느 점포(본점/무역점/신촌/천호)를 가나 = 위치 앵커" → 점포 분포·엔트로피·주력점포.
- **axiscombo**: "점포×시간대×결제×팀 조합이 카드주인 행동지문" → 축 교차 범주.
- **basket**: "한 영수증에 남녀 상품이 섞이면 가족 동반구매" → 영수증 단위 혼합도·순도(가족노이즈 탐지).
- **transition**: "백화점 안 동선(코너 방문 순서)이 성별 다르다" → 코너 bigram TF-IDF→SVD.
- **mission**: "한 번의 방문에 목적(미션)이 있다(장보기/쇼핑/선물)" → 영수증 미션 클러스터.

## F. 지도 타깃인코딩 / 세그먼트 (성별을 직접 사용, fold-safe) — ⚠️누수주의
- **teprec**: "각 카테고리의 성별 평균을 직접 인코딩" → team/part/buyer/pc/corner 성별 TE(금액·빈도 가중).
- **crosste**: "맥락별로 성별 평균이 다르다" → corner×습관, pc×주말, brd×점포 교차 TE.
- **supseg** ⭐(+0.0010, 그래프계열 최고): "비슷한 고객 군집마다 성별 경향이 있다" → 임베딩 군집(K30~600) fold-safe 성별 TE. *(비지도 임베딩에 지도 한 스푼 → 유일하게 살아난 방향)*
- **codeprefix / codedeep**: "상품코드(goodcd 13자리) 계층마다 성별성" → 앞 2~7자리 prefix 성별 log-odds.
- **hier**: 코드 계층 성별점수 통합(분포·분위).
- **xcat**: "카테고리(의류/식품/잡화) × 맥락(시간·요일·결제·수입) 교차 소비" → 35개 교차 지출비중.
- **rareW**: "희귀 상품에 가중한 성별 점수"(희귀구매가 더 변별적이라는 가설).

## G. 비지도 임베딩 / 표현학습 — 직교 다양성용 (대부분 ≈0, 취향 0.66 천장 재추출)
- **embed**: 고객×goodcd(1만) TF-IDF→SVD32.
- **brd_svd2**: 고객×brd(1882) TF-IDF→SVD32.
- **nmf_pc / nmf_corner**: 고객×카테고리 NMF 토픽(비음수 = 취향 토픽).
- **i2v / emb2 / embAll**: item2vec(상품 동시구매 임베딩) 계열.
- **comptok**: 상품명 구성토큰 임베딩.
- **ft**: FastText 상품/브랜드명 임베딩.
- **lda**: LDA 토픽(고객=문서, 상품=단어).
- **node2vec**: 고객-상품 이분그래프 랜덤워크 임베딩.
→ 공통 가설: "비지도 표현이 요약통계가 못 잡는 취향 군집을 잡는다." 실측: 거의 포화(같은 취향 신호 재추출). 가치는 앙상블 직교성뿐.

## H. 다양성 · 신선도
- **fresh**: "신상품·신선도 추구 성향 + 초기/최근 구매의 성별성" → 신상 히스토그램 + 첫/마지막 구매 성별점수 + SKU 반복/집중.

## I. 세션1 추가 (session1_FE/blocks)
- **block_s1_cross** ⭐(+0.0011, mega 위): "라벨=카드주인이라, 습관과 반대방향(교차) 구매는 선물 → 취향을 거꾸로 읽으면 새 신호" → corner 교차비중·교차취향·고가일회성(sharp). *(crossmulti와 같은 뿌리, corner 단일레벨 정밀판)*
- **block_s1_text**: "상품명을 글로 읽으면(언어모델) 요약통계와 다른 취향축" → 고객 구매 내러티브 문서 → 한국어 SBERT → SVD48. (약함 0.651, 텍스트축 직교 다양성)

## 한눈 요약 — 살아남은 가설 vs 죽은 가설
- ⭐ **살아남음(지도·수작업)**: 취향다양성(taxonomy) / 카테고리×가격(pricecat) / 선물타이밍(season) / **선물역전(cross·crossmulti)** / **지도세그먼트(supseg)** / 코드계층TE(codeprefix).
- ≈0 **포화/죽음(주로 비지도)**: 임베딩·그래프·토픽(node2vec·ALS·item2vec·LDA·NMF·SVD) 전부 — "취향=성별 0.66 천장"을 다른 형태로 재추출할 뿐.
- 결론: **"라벨이 카드주인"이라는 점을 파고든 지도 피처(선물 역전·세그먼트·가격·시즌)만 신호를 추가**했고, 비지도 딥은 다양성 보조역할.
