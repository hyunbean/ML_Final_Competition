7가지 기법
2.1Binning — 수치 피처 이산화적합

나이 45를 "40대", 혈압 132를 "130~139 구간"처럼 구간으로 묶는다. 같은 피처를 qcut 분위수 cut 등간격 반올림(Age÷5) 세 가지 방식으로 각각 변환해 서로 다른 분기 구조를 만든다. 트리가 "45살"과 "46살"을 다르게 처리하는 대신 같은 그룹으로 묶어 일반화를 돕는다.

2.2Digit Features — 자릿수 추출부적합

나이 45에서 십의 자리 "4"와 일의 자리 "5"를 별도 피처로 만든다. 합성 데이터는 특정 자릿수 패턴(끝자리 0이나 5 같은 반올림 흔적)을 담을 수 있어 유효하다.

2.3전체 피처 범주형 변환부적합

나이 45를 문자열 "45"로 변환한다. CatBoost/LGBM은 범주형에 ordered target encoding 등 다른 내부 처리를 해서 수치 버전과 다른 분기 패턴을 만든다.

2.4Frequency Encoding — 빈도 인코딩적합

나이 45인 사람이 전체 데이터에 500명 있다면, 해당 행에 500이라는 값을 추가한다. 희귀한 값을 가진 사람은 그 희귀함 자체가 질병 예측에 신호가 될 수 있다.

2.5Genetic Programming — 유전 프로그래밍부분 가능

gplearn이 "나이 × 콜레스테롤 ÷ 혈압" 같은 수식을 자동 생성한다. 개별 성능 향상보다 앙상블 다양성 확보가 주목적이다.

2.6원본 데이터 통계 추출적합 (더 강력)

대회 데이터가 원본에서 합성됐으므로 원본의 그룹별 통계(Target mean, Smoothed mean, WoE, Entropy)를 추출해 피처로 추가한다.

2.7DVAE — 노이즈 제거 변분 오토인코더적합 (오히려 더 유리)

피처에 노이즈를 추가하고 원래 값을 복원하도록 학습. 인코더가 피처 간 비선형 관계를 압축한 잠재 벡터를 만든다. SVD가 선형 압축이라면 DVAE는 비선형 압축이다.

2-Level Stacking — 10 Pattern Ridge Ensemble

이 솔루션은 단순한 Ridge 1개가 아니라 Optuna를 10번 다르게 실행해 매번 다른 OOF 조합을 뽑고, 각각 Ridge를 학습한 뒤 평균하는 구조다. L2 메타 모델 자체에도 다양성 원칙을 적용한 것이다.

핵심은 L1 모델들이 학습에 사용되지 않은 데이터에 대해 예측한 OOF(Out-of-Fold) 확률값을 L2 모델의 입력으로 쓴다는 것이다. 학습 데이터로 만든 예측을 그대로 쓰면 과적합이 발생하기 때문에 OOF가 필수적이다.

왜 Optuna를 10번 돌리는가
Optuna 탐색 자체도 확률적이다

Optuna는 내부적으로 확률적 탐색을 하기 때문에, 같은 100+ OOF 풀에서 출발해도 seed나 탐색 순서를 다르게 하면 매번 조금씩 다른 조합에 수렴한다. 10번 실행하면 조합 A, B, C … J가 만들어진다.

실행 #1 (seed=0) → [OOF-3, OOF-17, OOF-42 …] → Ridge A
실행 #2 (seed=1) → [OOF-5, OOF-17, OOF-38 …] → Ridge B
실행 #3 (seed=2) → [OOF-1, OOF-22, OOF-55 …] → Ridge C
…
실행 #10 (seed=9) → [OOF-9, OOF-31, OOF-44 …] → Ridge J
→ A+B+C+…+J 평균 = 최종 예측

어떤 한 번의 Optuna 실행 결과를 맹신하지 않고, 10번 평균함으로써 선택 과정의 분산을 줄이고 더 안정적인 메타 모델을 만든다. L1에서 다양성을 확보한 것과 같은 원칙을 L2 구성 방식에도 적용한 것이다.

각 폴드에서 해당 구간을 학습에 전혀 사용하지 않은 채 예측 → 5개 폴드 합치면 전체 학습 데이터에 대한 OOF 예측 완성

이 OOF 예측이 L2(Ridge)의 학습 데이터가 된다. 각 샘플은 자신을 학습에 쓰지 않은 모델이 예측한 값이므로 leakage가 없다.

Optuna 서브셋 탐색 — 왜 150개를 다 쓰지 않는가
전체 평균이 아닌 선별이 필요한 이유

150개 OOF를 Ridge에 모두 투입하면 오히려 성능이 떨어진다. 비슷한 모델끼리는 같은 방향으로 틀리기 때문에 숫자만 늘어날 뿐이다. Optuna로 2,500번의 시도를 통해 "서로 다른 실수를 하는 조합"을 탐색했고, 결과적으로 패턴당 ~20개씩 선별해 각 Ridge에 투입한다. 150개를 모두 쓰는 것이 아니라 약 1/7~1/8 수준으로 압축하는 것이다.

L2 메타 모델 — 왜 분류 문제에 Ridge(회귀)인가
Ridge 입력이 확률값이므로 구조적으로 회귀다

Ridge의 입력은 선택된 ~15개 L1 모델의 OOF 확률값(0~1 사이 실수)이다. 이 확률들을 선형 결합해 최종 확률을 만드는 것은 구조적으로 회귀 문제다.

L1 입력: [XGB확률, LGBM확률, Cat확률, ...]
          = [0.82,       0.79,       0.85,      ...]
L2 Ridge: w₁×0.82 + w₂×0.79 + w₃×0.85 + ...
최종 출력: 0.83 → 최종 예측 확률

AUC는 순위 기반 지표라 출력이 [0,1]을 벗어나도 순서가 같으면 AUC가 동일하다. Logistic Regression처럼 sigmoid를 씌울 필요가 없다.

Ridge가 유리한 이유

L2 정규화가 ~15개의 고상관 확률값들을 균등하게 처리한다. 비슷한 모델들 사이에서 한쪽으로 계수가 쏠리는 것을 막아 안정적인 가중 평균을 만든다.

비선형 메타 모델이 실패한 이유

XGBoost 같은 비선형 L2 모델은 자유도가 너무 높다. "fold 3에서만 잘 맞는 조합"을 진짜 신호로 착각해 과적합됐다.

선택이 생성만큼 중요하다
좋은 모델을 많이 만드는 것보다, 서로 다른 실수를 보완하는 모델을 골라내는 것이 결정적이었다. 150개를 만들고 15개를 쓰는 것이 150개를 모두 쓰는 것보다 강하다.
가장 중요한 교훈
가장 높은 CV(0.955865)를 얻었지만 최종 제출로 선택하지 않았다. 이 판단이 1등을 결정지었다. "CV를 신뢰하라"가 아니라 "CV–LB 관계를 신뢰하라"가 올바른 원칙이다.
CV 구간별 CV–LB 동작 패턴
분할 과적합(Split Overfitting)이란
비유로 이해하기

5-fold CV를 쓴다면 데이터가 5개의 폴드로 나뉜다. 모델은 4개 폴드로 학습하고 나머지 1개로 평가한다. 이 과정에서 각 폴드에만 있는 우연한 패턴을 실수로 학습하는 것이 분할 과적합이다.

예를 들어 fold 3의 검증 데이터에 우연히 고령 남성이 많이 모였다면, 모델은 "fold 3을 잘 맞추는 방향"으로 학습된다. 이는 진짜 신호가 아니라 데이터 분할의 우연한 결과다. CV는 올라가지만 실제 test 데이터에서는 그 우연한 패턴이 없으므로 LB는 오르지 않는다.

실제 시각화
각 행은 하나의 fold. 노란색 = 검증 폴드, 빨간색 = 과적합된 검증 폴드

앙상블을 계속 추가하다 보면 각 fold의 검증 성능을 높이기 위해 폴드별 특이 패턴에 맞추기 시작한다. 전체 CV는 올라가지만 test에서는 그 패턴이 없으므로 LB가 오르지 않거나 오히려 내려간다.

왜 발생하는가
정상 상태

CV가 오르면 LB도 비슷하게 오른다. 모델이 진짜 패턴을 더 잘 학습한 것.

분할 과적합 상태

CV는 오르는데 LB는 제자리거나 내려간다. 폴드 배치의 우연을 학습한 것.

최종 제출은 CV 0.95578~0.95580 구간에서 선택했다. 가장 높은 CV가 아닌, CV–LB 관계가 건강하게 유지되는 마지막 지점을 기준으로 삼았다.

KGMON Playbook 4단계 워크플로우
Step 1LLM 에이전트가 EDA 수행

LLM에게 train.csv / test.csv를 탐색하도록 지시. 행/열 수, 타깃 컬럼 형식, 범주형/수치형 구분, 결측값 존재 여부를 파악. LLM이 데이터 구조를 이해해야 다음 단계의 코드를 올바르게 생성할 수 있다.

"Please write EDA code to explore train.csv and test.csv. I will run the code and share the results back with you."
Step 2다양한 베이스라인 빠르게 구축

XGBoost, LightGBM, CatBoost, TabNet, Neural Net 등 다양한 모델을 각각 kfold로 학습. 모든 실험의 OOF와 Test 예측을 .npy 파일로 저장. 이 파일들이 이후 스태킹의 재료가 된다.

Step 3LLM이 피처 엔지니어링을 반복 수행

EDA 결과를 LLM에게 주고 "5가지 피처 아이디어를 제안하고 코드를 작성하라"고 반복 지시. GPU(cuDF/cuML)로 실험 속도를 높여 수백 개의 피처 아이디어를 단시간에 검증. 좋든 나쁘든 모든 실험의 OOF를 저장.

"Analyze the EDA output, suggest 5 feature engineering ideas, write the code. Repeat until CV = 0.92."
Step 4모델 결합 — Hill Climbing + 4-Level Stacking

수백 개의 OOF 파일을 LLM이 요약하고, 최적 조합을 Hill Climbing과 Ridge/Logistic Regression으로 탐색한다. 최종적으로 4-Level 스택을 구성한다.

Level 1 — 피처 추출 (GBDT/NN 다양한 피처셋)
↓
Level 2 — 패턴 추출 (L1 OOF → 메타 GBDT/NN)
↓
Level 3 — 패턴 정제 (L2 OOF → 메타 GBDT/NN)
↓
Level 4 — 최종 예측 (Ridge / Logistic Regression)
Hill Climbing이란
가장 강한 모델부터 시작해 탐욕적으로 추가

가장 OOF AUC가 높은 단일 모델로 시작한다. 나머지 모델을 하나씩 추가해보고 성능이 향상될 때만 유지한다. 성능이 오르지 않으면 버린다. 이를 더 이상 향상이 없을 때까지 반복. Optuna처럼 조합 탐색을 하되 더 단순하고 직관적이다.

현재 앙상블: [모델 A] → AUC 0.9540
+ 모델 B 추가 → AUC 0.9548 ✓ 유지
+ 모델 C 추가 → AUC 0.9545 ✗ 제거
+ 모델 D 추가 → AUC 0.9551 ✓ 유지
→ 최종: [A, B, D]

S6E3에서는 음수 가중치도 허용. 최종 두 제출은 Hill Climbing 결과와 Ridge/Logistic Regression 앙상블을 병행해 선택했다.

Pseudo-labeling & Extra Training
Test 데이터를 학습 신호로 활용

Pseudo-labeling: 앙상블 모델로 test 데이터를 예측해 고신뢰 샘플에 레이블을 붙이고 train에 추가 후 재학습. 또는 모든 test OOF를 pseudo-label로 활용해 새로운 단일 모델에 knowledge distillation을 수행한다.

Extra Training: CV 완료 후 전체 train 데이터(검증 폴드 포함)로 재학습. n_estimators를 CV 평균 best iteration × 1.2~1.3 배로 설정. 더 많은 데이터로 학습하므로 일반화 성능이 향상된다.

"Train a new NN or GBDT using knowledge distillation from all our OOF and Test PREDs and make a high performing single model."
S6E2가 "신중한 선택과 단순한 앙상블"이라면, S6E3는 "LLM으로 탐색 공간 자체를 압도적으로 넓힌 뒤 선택"하는 전략이다. 핵심 원칙(다양성 → 선택 → 스택)은 동일하지만 규모가 수십 배 다르다.
